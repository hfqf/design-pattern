# 依赖倒置原则

### **介绍**
依赖倒置原则（Dependence Inversion Principle，DIP）是指设计代码结构时，高层模块不应该依赖低层模块，二者都应该依赖其抽象。即面向接口编程。

抽象不应该依赖细节，细节应该依赖抽象。通过依赖倒置，可以减少类与类之间的耦合性，提高系统的稳定性，提高代码的可读性和可维护性，并且能够降低修改程序所造成的风险。

### **问题来源**
随着业务的扩展，增加功能就需要从底层实现到高层调用依次地修改代码。

### **优点**

1.不同模块解藕。

2.有效减小控制变动影响范围。

3.可形成插件化开发思路
